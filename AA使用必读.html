飞行雪绒 Script 开发指南（按当前代码）

1. 启动与自动发现
- 主入口：`lib/script/main.py`
- 动态发现：`lib/core/plugin_registry.py`
  - 管理器扫描：`lib/script/obj-*/manager.py`
  - 粒子扫描：`lib/script/practical/*_particle.py`
- 初始化流程：
  1) `discover_all()`
  2) `init_all_managers(pet)`
  3) 管理器在 `cleanup_all_managers()` 中统一清理

2. 管理器开发规范
文件位置
- 推荐：`lib/script/obj-你的模块/manager.py`

必备要素
- 继承 `BaseManager`
- 定义：`MANAGER_ID / DISPLAY_NAME / COMMAND_TRIGGER / COMMAND_HELP`
- 实现：`@classmethod create(...)`、`cleanup()`
- 在 `cleanup()` 里完整 `unsubscribe`，并关闭已创建窗口/对象
- 文件末尾注册：`manager_registry.register(YourManager.MANAGER_ID, YourManager)`

建议实践
- # 命令提示：在 `__init__` 调用 `get_hash_cmd_registry().register(...)`
- 跨管理器通信优先用事件：
  - 生成：`MANAGER_SPAWN_REQUEST`
  - 查询：`MANAGER_QUERY_REQUEST / MANAGER_QUERY_RESPONSE`
  - 目标查询：`TARGET_POSITION_QUERY / TARGET_POSITION_RESPONSE`
  - 保护检测：`PROTECTION_CHECK / PROTECTION_RESPONSE`

当前已内置管理器（obj-*）
- `snow_leopard`（#雪豹）
- `snow_pile`（#雪堆）
- `sofa`（#沙发 / #沙发重力）
- `mortor`（#摩托）
- `clock`（#闹钟 / #闹钟重力）
- `speaker`（#音响 / #音响重力 / #退出音乐登录）

补充处理器（非 obj 目录）
- `lib/script/practical/cleanup_handler.py`：注册 `#清理`

3. 粒子脚本开发规范
文件位置
- 推荐：`lib/script/practical/xxx_particle.py`

必备要素
- 继承 `BaseParticleScript`
- 使用 `@register_particle("particle_id")`
- 定义 `PARTICLE_ID`
- 实现 `create_particles(area_type, area_data)`

粒子实例约定（被 `ParticleOverlay` 渲染）
- 必需：`update()`、`alive`（属性或可调用）
- 生命周期字段：`life`、`max_life`（建议提供，便于统一淡出）
- 可选绘制标记：
  - `is_text=True`（文字）
  - `is_line=True`（线段）
  - `is_circle=True`（圆）
  - `no_fade=True`（不参与透明度淡出）
  - `width/height`（矩形分支）

区域参数约定
- `point`: `(x, y)`
- `rect`: `(x1, y1, x2, y2)`
- `circle`: `(x, y, radius)`

4. 实体接口（PetWindow / BaseEntity）
- 状态切换：`change_state(state)`
- 播放动画：`play_animation(state, duration=0)`
- 调度任务：`schedule_task(callback, delay_ms, repeat=False)`
- 取消任务：`cancel_task(task_id)`
- 粒子：`spawn_particles(x, y, particle_id='scatter_fall', area_type='point', area_data=None)`

注意
- 当前默认粒子 ID 是 `scatter_fall`（不是历史的 `snowflake`）
- 建议显式传入粒子 ID，避免默认值变化影响行为

5. 输入链路与命令分发
命令输入框：`lib/script/ui/command_dialog.py`
- `/xxx` -> 发布 `INPUT_COMMAND`（由 `CmdCenter` 执行 shell）
- `#xxx` -> 发布 `INPUT_HASH`（由各管理器/处理器订阅）
- 普通文本 -> 发布 `INPUT_CHAT`（由 `ChatHandler` 处理）

AI 工具调度
- `ToolDispatcher` 监听 `STREAM_FINAL`
- 从模型回复中提取 `###指令###` / `###指令 参数###`
- 当前支持：音乐、下一曲、暂停、雪豹、沙发、摩托、闹钟/计时、音量、瞬移、回忆

6. 最小管理器模板
```python
from lib.core.plugin_registry import BaseManager, manager_registry
from lib.core.event.center import get_event_center, EventType

class DemoManager(BaseManager):
    MANAGER_ID = "demo"
    DISPLAY_NAME = "示例管理器"
    COMMAND_TRIGGER = "示例"
    COMMAND_HELP = "[数量] - 示例命令"

    def __init__(self, entity=None):
        self._entity = entity
        self._ec = get_event_center()
        self._ec.subscribe(EventType.INPUT_HASH, self._on_hash)

    @classmethod
    def create(cls, entity=None, **kwargs):
        return cls(entity)

    def _on_hash(self, event):
        pass

    def cleanup(self):
        self._ec.unsubscribe(EventType.INPUT_HASH, self._on_hash)

manager_registry.register(DemoManager.MANAGER_ID, DemoManager)
```

7. 最小粒子模板
```python
from lib.script.practical.base_particle import BaseParticleScript
from lib.core.plugin_registry import register_particle

@register_particle("demo_particle")
class DemoParticleScript(BaseParticleScript):
    PARTICLE_ID = "demo_particle"

    def create_particles(self, area_type, area_data):
        x, y = area_data[0], area_data[1]
        return [DemoParticle(x, y)]

class DemoParticle:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        self.life = 1.0
        self.max_life = 1.0

    def update(self):
        self.life -= 0.05

    @property
    def alive(self):
        return self.life > 0
```

事件系统使用说明

飞行雪绒事件系统使用说明（按当前代码）

1. 核心机制
实现位置
- `lib/core/event/center.py`
- 全局入口：`get_event_center()`（单例）

处理模型
- `publish(Event)` -> 入队 -> 按订阅顺序依次回调
- 主线程发布：若当前未处理队列，立即 drain
- 非主线程发布：通过 Qt signal 回主线程再处理
- 防长循环保护：单次最多处理 100 个事件，剩余事件下一轮继续

中断机制
- 回调中可调用 `event.mark_handled()`
- 同一事件类型后续订阅者将被跳过

2. 常用用法
```python
from lib.core.event.center import get_event_center, EventType, Event

ec = get_event_center()

def on_event(event: Event):
    data = event.data

ec.subscribe(EventType.TICK, on_event)
ec.publish(Event(EventType.INFORMATION, {'text': 'hello'}))
ec.unsubscribe(EventType.TICK, on_event)
```

3. 输入事件链路（实际行为）
鼠标
- Qt 原始事件 -> `ClickHandler` -> 发布：
  - `MOUSE_PRESS`（按下即发）
  - `MOUSE_MOVE`
  - `MOUSE_ENTER` / `MOUSE_LEAVE`
  - `MOUSE_CLICK`（双击窗口超时后确认单击）
  - `MOUSE_DOUBLE_CLICK`（双击窗口内第二次点击立即确认）

键盘
- Qt 原始事件 -> `KeyHandler` -> `KEY_PRESS` / `KEY_RELEASE`

命令框回车
- `/xxx` -> `INPUT_COMMAND`
- `#xxx` -> `INPUT_HASH`
- `xxx` -> `INPUT_CHAT`

4. 关键事件协议（高频使用）
INFORMATION
- `EventType.INFORMATION`
- 常用字段：
  - `text: str`
  - `min: int`（最少显示 tick）
  - `max: int`（最多显示 tick）
  - `align: 'left'|'center'`（可选）
  - `particle: bool`（可选）

PARTICLE_REQUEST
- `EventType.PARTICLE_REQUEST`
- 字段：
  - `particle_id: str`
  - `area_type: 'point'|'rect'|'circle'`
  - `area_data: tuple`
- 坐标使用全局坐标，`ParticleOverlay` 内部转本地坐标

MANAGER_SPAWN_REQUEST
- 请求某管理器生成对象
- 常见字段：
  - `manager_id: 'snow_leopard'|'snow_pile'|'sofa'|'mortor'|'clock'|'speaker'`
  - `count: int`（可选）
  - `spawn_type: str`（如 `natural` / `command`）
  - 其他管理器自定义字段（如闹钟 `seconds`）

MANAGER_QUERY_REQUEST / RESPONSE
- 跨管理器查询信息（数量、位置、保护范围等）
- 建议携带 `request_id` 做请求-响应配对

TARGET_POSITION_QUERY / RESPONSE
- 统一目标位置查询（解耦状态机与具体管理器）

PROTECTION_CHECK / RESPONSE
- 主宠保护半径检测，支持多管理器聚合响应

ENTITY_POSITION_REQUEST / RESPONSE
- 查询实体（当前主要是 `pet_window`）位置与尺寸

ENTITY_STATE_QUERY / RESPONSE
- 查询实体移动状态/当前状态

PET_TELEPORT
- 主宠物瞬移事件
- 支持：
  - `x, y`
  - `position: QPoint|(x,y)`

5. 音乐与聊天事件
音乐
- 播放控制：`MUSIC_PLAY_TOP`、`MUSIC_ENQUEUE`、`MUSIC_NEXT_TRACK`、`MUSIC_PLAY_PAUSE`
- 队列操作：`MUSIC_PLAY_QUEUE_INDEX`、`MUSIC_ENQUEUE_HISTORY`、`MUSIC_ENQUEUE_LIKED`、`MUSIC_ENQUEUE_LOCAL`
- 进度：`MUSIC_PROGRESS_REQUEST` -> `MUSIC_PROGRESS`
- 登录：`MUSIC_LOGIN_REQUEST` / `MUSIC_LOGOUT_REQUEST` / `MUSIC_LOGIN_STATUS_CHANGE`

聊天
- 输入：`INPUT_CHAT`
- 流式最终文本：`STREAM_FINAL`
- `ToolDispatcher` 订阅 `STREAM_FINAL` 解析 `###工具指令###`

6. 生命周期事件
- `APP_PRE_START`：预启动（发现插件、启动延时逻辑）
- `APP_INIT_READY`：初始化就绪
- `APP_START`：进入启动阶段
- `APP_MAIN`：主循环就绪
- `APP_EXIT`：退出清理
- `APP_QUIT`：应用退出请求

7. 开发约束
- 回调必须短小，避免阻塞 UI 线程
- 每个 `subscribe` 都要有对称 `unsubscribe`（通常在 `cleanup`）
- `mark_handled()` 仅用于需要阻断传播的场景
- 事件字段命名保持稳定，避免同语义多套 key

飞行雪绒粒子效果说明（当前注册项）

1. 总览
入口与渲染
- 事件入口：`EventType.PARTICLE_REQUEST`
- 覆盖层：`lib/core/qt_particle_system.py`（`ParticleOverlay`）
- 脚本管理：`lib/script/practical/manager.py`
- 自动发现：`discover_particles()` 扫描 `lib/script/practical/*_particle.py`

全局配置（`config/config.py`）
- `PARTICLES.enable_stroke`：是否描边
- `PARTICLES.fade_threshold`：开始淡出的生命比例阈值

注意
- 当前注册粒子共 14 个
- 默认粒子 ID 是 `scatter_fall`
- 历史 ID（如 `snowflake`、`heart`）当前未注册，不可用

2. 已注册粒子（14个）

[1] `scatter_fall`
- 文件：`lib/script/practical/scatter_fall_particle.py`
- 形态：浅青色方块
- 行为：四向扩散，阻力 + 重力下落
- 触发：主宠动作 `jumping` 开始（`Actions.JUMPING`）

[2] `pink_scatter_fall`
- 文件：`lib/script/practical/pink_scatter_fall_particle.py`
- 形态：浅粉色方块
- 行为：与 `scatter_fall` 同模型
- 触发：
  - 主宠右键按下（`MouseEventHandler`）
  - 主宠动作 `happy` 开始（`Actions.HAPPY`）

[3] `cyan_pink_scatter_fall`
- 文件：`lib/script/practical/cyan_pink_scatter_fall_particle.py`
- 形态：青/粉随机混色方块
- 行为：与 `scatter_fall` 同模型
- 触发：主宠左键按下（`MouseEventHandler`）

[4] `snow`
- 文件：`lib/script/practical/snow_particle.py`
- 形态：白色圆形雪粒（`is_circle=True`）
- 行为：全向发射，阻力 + 重力，随生命周期淡出
- 触发：
  - 雪豹淡出消失
  - 雪堆触发生成雪豹时的出现特效

[5] `snow_drift`
- 文件：`lib/script/practical/snow_drift_particle.py`
- 形态：白色圆形雪粒（`is_circle=True`）
- 行为：下落后在底部短暂停留堆积，再缓慢消退
- 触发：
  - 雪豹跳跃开始与落地弹跳
  - 雪堆左键、右键、双击交互

[6] `burst_line`
- 文件：`lib/script/practical/burst_line_particle.py`
- 形态：彩色线段（`is_line=True`，`no_fade=True`）
- 行为：中心爆发，匀减速，线长随寿命缩短
- 触发：
  - 雪豹单击确认（触发跳跃）
  - 主宠瞬移时的原地爆发效果

[7] `right_fade`
- 文件：`lib/script/practical/right_fade_particle.py`
- 形态：浅青色横向小矩形
- 行为：向右漂移并淡出
- 触发：多处 UI 淡出/关闭（命令框、气泡、按钮、播放列表、搜索框等）

[8] `up_fade`
- 文件：`lib/script/practical/up_fade_particle.py`
- 形态：粉色方块（小概率黑色变体）
- 行为：向上移动并淡出

飞行雪绒调度系统使用说明（按当前代码）

1. 调度器结构
实现位置
- `lib/core/timing/manager.py`
- 全局访问注册：`lib/core/timing/__init__.py`

核心类
- `TimingManager(QObject)`
- 三个独立 QTimer：
  - `TICK`：固定 50ms（20Hz）
  - `FRAME`：`1000 // frame_fps` ms
  - `GIF_FRAME`：`1000 // gif_fps` ms

当前默认值（由 `config/config.py` 提供）
- `ANIMATION['frame_fps'] = 60`
- `ANIMATION['gif_fps'] = 16`

2. 事件发布
`TimingManager` 周期发布：
- `EventType.TICK`（`tick_count`）
- `EventType.FRAME`（`frame_count`）
- `EventType.GIF_FRAME`（`frame_count`）

任务到时发布：
- `EventType.TIMER`
  - `task_id`
  - `repeat`

3. 任务 API
底层接口
- `add_task(interval_ms, repeat=True) -> task_id`
- `remove_task(task_id)`
- `pause_task(task_id)` / `resume_task(task_id)`
- `clear_all()`

实体层推荐接口（PetWindow）
```python
task_id = pet.schedule_task(callback, delay_ms=1000, repeat=False)
pet.cancel_task(task_id)
```

实际执行链路
1. `schedule_task()` 调用 `TimingManager.add_task()`
2. `PetWindow` 将 `task_id -> callback` 存入 `_task_callbacks`
3. 到时收到 `TIMER` 事件后执行 callback
4. 非重复任务执行后自动移除 callback 映射

4. 时间精度与换算
内部换算
- `interval_ticks = max(1, int(interval_ms / 50))`

含义
- 所有任务按 50ms tick 对齐
- 小于 50ms 会被提升为 1 tick（50ms）
- 非 50ms 整数倍会向下取整

示例
- `1000ms -> 20 ticks`
- `120ms -> 2 ticks`（约 100ms）
- `30ms -> 1 tick`（50ms）

5. 任务暂停机制（全局）
事件
- `TIMER_PAUSE`
- `TIMER_RESUME`

行为
- 只暂停“任务系统”（`TIMER` 任务），不暂停 `TICK/FRAME/GIF_FRAME`
- 支持多来源引用计数：
  - `TIMER_PAUSE` 可携带 `source`
  - 必须对应 `TIMER_RESUME(source)` 才会解除该来源的暂停
- 典型场景：命令框显示时暂停漫游任务，关闭后恢复

6. FPS 动态调整
- `set_frame_fps(fps)`：范围钳制 1~120
- `set_gif_fps(fps)`：范围钳制 1~60
- 运行中调用会即时 `setInterval`

7. 开发建议
- 业务逻辑优先订阅 `TICK/FRAME`，不要自行创建并行 QTimer
- 高频重计算尽量避开 `FRAME`
- 在 `cleanup` 中移除任务并退订事件，避免残留回调
- 需要更高精度时请明确说明“tick 对齐误差”对业务是否可接受
