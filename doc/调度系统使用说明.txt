飞行雪绒调度系统使用说明（按当前代码）

1. 调度器结构
实现位置
- `lib/core/timing/manager.py`
- 全局访问注册：`lib/core/timing/__init__.py`

核心类
- `TimingManager(QObject)`
- 三个独立 QTimer：
  - `TICK`：固定 50ms（20Hz）
  - `FRAME`：`1000 // frame_fps` ms
  - `GIF_FRAME`：`1000 // gif_fps` ms

当前默认值（由 `config/config.py` 提供）
- `ANIMATION['frame_fps'] = 60`
- `ANIMATION['gif_fps'] = 16`

2. 事件发布
`TimingManager` 周期发布：
- `EventType.TICK`（`tick_count`）
- `EventType.FRAME`（`frame_count`）
- `EventType.GIF_FRAME`（`frame_count`）

任务到时发布：
- `EventType.TIMER`
  - `task_id`
  - `repeat`

3. 任务 API
底层接口
- `add_task(interval_ms, repeat=True) -> task_id`
- `remove_task(task_id)`
- `pause_task(task_id)` / `resume_task(task_id)`
- `clear_all()`

实体层推荐接口（PetWindow）
```python
task_id = pet.schedule_task(callback, delay_ms=1000, repeat=False)
pet.cancel_task(task_id)
```

实际执行链路
1. `schedule_task()` 调用 `TimingManager.add_task()`
2. `PetWindow` 将 `task_id -> callback` 存入 `_task_callbacks`
3. 到时收到 `TIMER` 事件后执行 callback
4. 非重复任务执行后自动移除 callback 映射

4. 时间精度与换算
内部换算
- `interval_ticks = max(1, int(interval_ms / 50))`

含义
- 所有任务按 50ms tick 对齐
- 小于 50ms 会被提升为 1 tick（50ms）
- 非 50ms 整数倍会向下取整

示例
- `1000ms -> 20 ticks`
- `120ms -> 2 ticks`（约 100ms）
- `30ms -> 1 tick`（50ms）

5. 任务暂停机制（全局）
事件
- `TIMER_PAUSE`
- `TIMER_RESUME`

行为
- 只暂停“任务系统”（`TIMER` 任务），不暂停 `TICK/FRAME/GIF_FRAME`
- 支持多来源引用计数：
  - `TIMER_PAUSE` 可携带 `source`
  - 必须对应 `TIMER_RESUME(source)` 才会解除该来源的暂停
- 典型场景：命令框显示时暂停漫游任务，关闭后恢复

6. FPS 动态调整
- `set_frame_fps(fps)`：范围钳制 1~120
- `set_gif_fps(fps)`：范围钳制 1~60
- 运行中调用会即时 `setInterval`

7. 开发建议
- 业务逻辑优先订阅 `TICK/FRAME`，不要自行创建并行 QTimer
- 高频重计算尽量避开 `FRAME`
- 在 `cleanup` 中移除任务并退订事件，避免残留回调
- 需要更高精度时请明确说明“tick 对齐误差”对业务是否可接受
